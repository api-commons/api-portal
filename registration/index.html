Self service registration is a trademark of the world of APIs, over earlier incarnations like SOA. APIs provide free, and paid access, 24/7, via the Internet.

This is just a demo site, so we don't have registration, but if we did we'd provide registration via:

  • Manually Create New Account
  • OAuth using Github
  • OAuth using Twitter
  • OAuth using LinkedIn
  • OAuth using Facebook

Even if I only get introductory access to an API, all developers should get immediate access to play with APIs, otherwise the chance I will adopt and put to use, go way down.

Self-service registration for API resources, even if they are private, are as critical as documentation, and code samples.
